# CRAP 四原则评估框架

## R — 原文 (Reading)

> "以下是对基本设计原则的概述，每一个优秀的设计中都应用了这些设计原则。尽管我在后面将逐个讨论各个原则，不过要记住，它们实际上是相互关联的。只应用某一个原则的情况很少。"
>
> — Robin Williams, 第 1 章

> "一旦能够说出什么东西的名字，就会很容易注意到它。你就会掌握它，拥有它，让它受你所控。"
>
> — Robin Williams, 第 1 章 (约书亚树)

---

## I — 方法论骨架 (Interpretation)

**CRAP 四原则**是把"看起来很业余"这个模糊感受拆解为 4 个可检查、可修正的维度。它不是"创造设计"的配方，而是**诊断和改进**的框架。

1. **亲密性 (Proximity)** — 物理距离 = 逻辑关系。相关的元素放一起，不相关的拉开。一张页面上眼睛停留的次数 = 孤立单元的数量，目标是 ≤5 次。
2. **对齐 (Alignment)** — 每个元素必须在视觉上与另一个元素"连上线"。左/右对齐优于居中对齐，因为硬边界更明确。居中对齐是初学者的默认选项，通常应避免。
3. **重复 (Repetition)** — 有意识地在整个作品中重复某个视觉要素(字体/颜色/线/符号/空间关系)。把无意的"一致"加强为有意的"品牌"。
4. **对比 (Contrast)** — 不相同的元素必须"截然不同"而非"略有差异"。12pt vs 14pt 是冲突不是对比。通过大小/粗细/颜色/形状/方向/结构六维度实现。

这四个维度按顺序逐一检查，每步只改一个原则，迭代推进。

## A1 — 书中的应用 (Past Application)

### 案例 1: 报告封面的四原则渐进改造
- **问题**: 一个典型的乏味报告封面——居中对齐、各行间距均匀、6 行孤立文字、没有层级。
- **方法论的使用**: 作者依次应用四个原则: ①亲密性(标题和子标题靠近，作者和日期远离)→②对齐(从左对齐替代居中对齐)→③重复(标题字体在作者名中重复)→④对比(增加黑方框+反色文字)。
- **结论**: 同样的基本信息，四步改造后封面从"业余"变为"专业精美"。
- **结果**: 每步只改一个原则，读者能看到每一步的差异。

### 案例 2: 企业名片的亲密性+对齐改造
- **问题**: 名片上 5 项内容(姓名/职位/公司/地址/电话)散落，读者眼睛停 5 次，不知道从哪里开始。
- **方法论的使用**: ①亲密性→相关项分组(姓名+职位一组，地址+电话一组)→②对齐→全部统一为右对齐→③重复→粗体在关键信息中重复→④对比→增大字体差异。
- **结论**: 仅通过距离调整+对齐统一，名片从"散乱"变为"专业"。
- **结果**: 读者一眼知道从哪里开始、到哪里结束。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. **做 PPT/文档时** — 幻灯片看起来很"素"或"乱"，说不上为什么。
2. **收到排版任务时** — 被要求设计传单、海报、简报、名片等，没有设计背景。
3. **评审他人作品时** — 需要给同事/下属的设计稿提出具体的修改意见，而不是笼统说"不够好看"。
4. **自己做个人项目时** — 简历、博客配图、社交媒体封面、活动邀请函。
5. **转译到新媒介时** — 把一篇纯文本内容做成图文版(微信公众号/Newsletter)。

### 语言信号 (用户的话里出现这些就应激活)

- "这个页面看起来很乱/业余/不专业"
- "帮我设计一个名片/传单/海报"
- "这个排版哪里不对"
- "把它弄好看一点"
- "this design looks off / unprofessional / amateur"
- "how can I make this layout better"

### 与相邻 skill 的区分

- 与 `type-contrast-six-dimensions` 的区别: CRAP 是**整体布局评估框架**，覆盖页面结构、空白、对齐等；字体对比是 CRAP 中"对比"维度在字体上的深入展开。当用户的问题是"这个字体组合感觉怪怪的"时，应调用字体对比 skill。
- 与 `center-point-focus` 的区别: 中心点框架是对比原则的具体化——当页面上所有元素无主次时调用中心点 skill；当需要全面检查页面时调用 CRAP。
- 与 `color-wheel-scheme` 的区别: 颜色是 CRAP 中对比和重复的重要工具。等用户明确提到颜色问题时调用色轮配色 skill，否则用 CRAP 框架先做全局诊断。
- 与 `pro-type-rules` 的区别: 排版规则是细节层面的执行标准(一个空格/印刷引号/不用下划线)。先 CRAP 诊断整体结构，再用排版规则打磨细节。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行：

1. **第一步：去芜存菁 — 去掉多余元素**
   - 完成标准: 已识别并移除所有非必要元素(边框、多余的标签如"电话""邮箱"、装饰性剪贴画)。页面只保留必须传达的信息。
   - 判停条件: 如果内容已是纯文本且无不必要装饰，跳过此步。

2. **第二步：检查亲密性 — 分组与空间**
   - 完成标准: 所有逻辑相关的元素已物理靠近形成视觉单元。不相关元素之间有明显更大的空间。眼睛在页面上停留的次数已减少到 ≤5。
   - 操作: 眯起眼睛，数一数页面上有多少个"视觉块"。如果你数出超过 5 个，就考虑合并。

3. **第三步：检查对齐 — 建立硬边界**
   - 完成标准: 页面上所有元素已统一为一种对齐方式(优先左对齐)。每个元素都能在页面上找到与之对齐的另一个元素。可以画出一条"看不见的线"贯穿它们。
   - 判停条件: 如果有意使用居中对齐(如婚礼邀请函等正式场合)，保留但需确认此选择是有意的。

4. **第四步：建立重复 — 强化一致**
   - 完成标准: 已找到至少一个可重复的视觉要素(颜色/字体/线/符号)，并在整个作品中至少出现 2 次。重复不能过度(不能所有元素都一样)。
   - 检查: 标题、项目符号、链接颜色、装饰元素中是否有重复可用。

5. **第五步：制造对比 — 突出中心点**
   - 完成标准: 确定了一个明确的视觉"中心点"(读者第一眼应该看哪里)，并通过大小/粗细/颜色使其与其他元素"截然不同"。页面上没有多个互相争夺注意力的要素。
   - 检查: 你能否在 2 秒内说出"这个页面最吸引我的是什么"？如果不能，对比还不够。

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **当内容是纯功能性文档(API 文档/法律条款/学术论文)时**——清晰和效率优先于视觉冲击，CRAP 的某些原则(如对比)要适度使用。
- **当用户明确要求"不要改动，只帮我检查内容"时**——不要主动设计。
- **当设计已经由专业设计师完成时**——不要用四个原则"挑刺"，除非用户明确要求评审。
- **当用户需要的是品牌 VI 系统设计时**——CRAP 框架可以辅助但不足以创建完整的品牌体系。

### 作者在书中警告的失败模式

- **重复过度**: 太多重复元素会让人厌烦。比如红色鞋子+红色口红+红色帽子+红色手包——不是统一，是过火。
- **对比不充分**: 12pt 和 14pt 的对比不是对比，是冲突。要么一样要么迥异。
- **截留空白**: 空白被夹在元素之间形成尴尬形状时，比没有空白更糟。
- **没有中心点**: 所有东西都很大/很粗/很华丽 → 没有任何东西突出。

### 作者的盲点 / 时代局限

- 本书以印刷品为中心(纸张/油墨/出血)，当代数字媒体(响应式/动效/交互)的许多场景需要扩展。
- "好看=有效"这个假设未论证——对某些场景(如找药信息、安全说明)，清晰比好看重要。
- 这些原则基于拉丁文字系统，中文字体排版的适配需要额外调整(特别是字体分类部分)。

### 容易混淆的邻近方法论

- "少即是多"极简主义——CRAP 不要求极简，只要求有组织。四原则可以用于繁复的设计(如维多利亚风格海报)。
